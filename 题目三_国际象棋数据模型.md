# 题目三：国际象棋比赛数据模型

| 实体 | 说明 |
|------|------|
| Clubs | 俱乐部 |
| Players | 棋手 |
| Members | 会员（棋手与俱乐部的关联） |
| Tournaments | 锦标赛 |
| Matches | 比赛 |

## 实体关系

- 俱乐部可以有很多会员（1:N）
- 一个棋手可以有一个且只有一个排名（1:1，排名可作为棋手属性）
- 俱乐部可以举办许多锦标赛（1:N）
- 锦标赛也可以由其他组织赞助（N:M，多对多）
- 每年都有许多锦标赛
- 棋手可以参加零次或多次锦标赛（N:M）
- 一个棋手在任何时候只能是一个俱乐部的会员（通过会员表实现，带时间约束）
- 一个锦标赛可以有许多棋手（N:M）
- 一个锦标赛可以有零场或多场比赛，每场比赛在两个参赛棋手之间进行（1:N，Match 关联两个 Player）

## 建表(SQL)

```sql
-- 1. 俱乐部表
CREATE TABLE clubs (
    club_id      VARCHAR(50) PRIMARY KEY,
    name         VARCHAR(200) NOT NULL,
    address      TEXT,
    details      TEXT,
    created_at   TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 2. 棋手表
CREATE TABLE players (
    player_id    VARCHAR(50) PRIMARY KEY,
    name         VARCHAR(200) NOT NULL,
    address      TEXT,
    details      TEXT,
    ranking      INTEGER UNIQUE,  -- 一个棋手有且只有一个排名
    created_at   TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 3. 会员表（棋手-俱乐部关联，一个棋手同一时间只能属于一个俱乐部）
CREATE TABLE members (
    member_id    SERIAL PRIMARY KEY,
    player_id    VARCHAR(50) NOT NULL REFERENCES players(player_id),
    club_id      VARCHAR(50) NOT NULL REFERENCES clubs(club_id),
    joined_at    DATE NOT NULL,
    left_at      DATE,  -- 可为空，表示当前仍在俱乐部
    UNIQUE (player_id, joined_at),  -- 同一棋手同一时间只能在一个俱乐部
    CHECK (left_at IS NULL OR left_at >= joined_at)
);

-- 4. 赞助方表（企业、政府等组织）
CREATE TABLE sponsors (
    sponsor_id   VARCHAR(50) PRIMARY KEY,
    name         VARCHAR(200) NOT NULL,
    type         VARCHAR(50),
    details      TEXT
);

-- 5. 锦标赛表
CREATE TABLE tournaments (
    tournament_code VARCHAR(50) PRIMARY KEY,
    name           VARCHAR(200) NOT NULL,
    start_date     DATE NOT NULL,
    end_date       DATE NOT NULL,
    host_club_id   VARCHAR(50) REFERENCES clubs(club_id),  -- 举办俱乐部，可为空
    CHECK (end_date >= start_date)
);

-- 6. 锦标赛-赞助方关联表（多对多，零个、一个或多个赞助方）
CREATE TABLE tournament_sponsors (
    tournament_code VARCHAR(50) REFERENCES tournaments(tournament_code),
    sponsor_id      VARCHAR(50) REFERENCES sponsors(sponsor_id),
    PRIMARY KEY (tournament_code, sponsor_id)
);

-- 7. 锦标赛-棋手参赛关联表（棋手可参加零次或多次锦标赛）
CREATE TABLE tournament_players (
    tournament_code VARCHAR(50) REFERENCES tournaments(tournament_code),
    player_id      VARCHAR(50) REFERENCES players(player_id),
    PRIMARY KEY (tournament_code, player_id)
);

-- 8. 比赛表（两个棋手之间的对局）
CREATE TABLE matches (
    match_id        SERIAL PRIMARY KEY,
    tournament_code VARCHAR(50) NOT NULL REFERENCES tournaments(tournament_code),
    player1_id      VARCHAR(50) NOT NULL REFERENCES players(player_id),
    player2_id      VARCHAR(50) NOT NULL REFERENCES players(player_id),
    match_date      DATE,
    result          VARCHAR(50),
    CHECK (player1_id != player2_id)
);
```

| 关系 | 实现方式 |
|------|----------|
| 俱乐部-会员 | `members` 表，club_id和player_id |
| 棋手-排名 | `players.ranking` 字段，UNIQUE 约束 |
| 俱乐部-锦标赛 | `tournaments.host_club_id` |
| 锦标赛-赞助方 | `tournament_sponsors` 多对多关联表 |
| 棋手-锦标赛 | `tournament_players` 多对多关联表 |
| 锦标赛-比赛 | `matches.tournament_code` |
| 比赛-棋手 | `matches.player1_id`, `matches.player2_id` |
